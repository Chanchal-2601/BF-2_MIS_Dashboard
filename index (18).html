Quick Start

1. Set the date in the top bar — every page updates to that day's data at once. Defaults to the most recent day the furnace was running.
2. Start on Executive Summary — it's the one-page answer to "how is BF-2 doing today," with production, fuel, and performance scores together.
3. Check Alerts & Insights next — if anything needs attention, it's flagged there with the reason and what to do about it, not just a number out of range.
4. Drill into detail pages (Raw Material, Process, Fuel, Monthly Review, Predictive Analytics, Resource Utilization) only for whatever the Alerts page or your own question points you toward.
5. Use Reports when you need a complete, printable daily sheet — e.g. for a physical file or a meeting handout.
Uploading new data or fixing something? Jump straight to Section 4 (Uploading DPR/MIS Files) or Section 5 (Manual Dataset Update) in this guide.

Dashboard Overview

Ten pages, each answering a different question about BF-2's operation.
📊 Executive Summary
The one-page daily snapshot: Production, Productivity, Coke Rate, Total Fuel Rate, HBT, O₂ Enrichment, plus IBRM composition, alerts, and monthly summary. Start here every day.
⛏️ Raw Material
Sinter, pellet and coke quality — Basicity, FeO%, Alumina, CCS, Ash%, Moisture% and more, which drive much of what happens downstream in the furnace.
⚙️ Process Parameters
The core operational readings for the day — Wind Volume, HBT, O₂ Enrichment, Oxygen Consumption, burden composition.
🔥 Fuel & Efficiency
Coke Rate, PCI Rate, Total Fuel Rate, Slag Rate, HBT and Hot Blast Pressure, plus fuel consumption trends and split.
📅 Monthly Review
Month-level averages, best/worst day, running-day count, and the Primary Constraint insight — which parameter caused the most deviations that month, and which department should act on it.
🚨 Alerts & Insights
Every deviation the Root Cause Analysis Engine finds, each with evidence, root cause, operational impact, corrective action, and responsible department. See Section 7.
🔮 Predictive Analytics
Trend-based 7-day forecasts, plus three operational risk probabilities: Stable Operation, Furnace Shutdown, and Process Deterioration.
🏭 Resource Utilization
BF Gas generation, consumption, and flaring; gas chemistry (CO/CO₂/H₂/ηCO); and granulated slag by-product recovery — all real measured data.
📋 Reports
The complete Daily MIS Report in one printable sheet — every parameter, organized into sections, ready to export or hand to management.
📥 Data Management
Where all the data above actually comes from — upload a DPR/MIS file, link a live Excel file, or edit the shared master dataset. See Sections 4–6.

Navigation & Controls

Everything visible on screen, explained.
Sidebar (left): pages are grouped by category — Operations (Executive Summary, Raw Material, Process, Fuel), Analysis (Monthly Review, Alerts, Predictive Analytics), Sustainability (Resource Utilization), Reports, and Data (Data Management). Click any item to switch instantly — the whole dashboard is one page underneath, so there's no reload.
Date picker (top bar): every day-level page — Executive Summary, Raw Material, Process, Fuel, Alerts — reads from whichever date is selected here. Change it once at the top; every page follows.
Shift filter (top bar): narrows day-level views to a specific shift (A/B/C) where shift-level data is available, or "All" for the full day.
Status badge (top bar): shows ● RUNNING (green, Production ≥ 3,000 MT), ● START-UP (amber, 1–2,999 MT — furnace ramping up after a shutdown), or ● SHUTDOWN (red, 0 MT) for the selected date — derived from that day's actual Production figure, not a live sensor feed.
Data range indicator (top bar, next to the clock): shows the full span of dates currently loaded, e.g. "Data: 01-01-2026 to 31-05-2026 (151 days)" — this grows automatically as new data is added, and tells you at a glance how current the dataset is.
Month selector (Monthly Review page only): switches which month's data that page summarizes — independent of the top-bar date picker.
❓ User Guide icon (top bar, far right): opens this guide. Close it with the ✕ button, the Escape key, or by clicking outside the panel.
Color legend, used throughout the dashboard:
GREEN within target / good BLUE informational / moderate deviation AMBER needs attention RED critical / requires action

There are no fixed cell positions — the engine reads every sheet and matches column headers (like "Production" or "Coke Rate") wherever they appear, even if rows or columns shift between reports, or headers carry units like "Production (MT)". It ignores logos, titles, signatures, remarks and blank rows automatically, since those never match a known parameter.
Handles messy real-world files: merged header cells, comma-formatted numbers ("9,800"), and stray unit text in a cell ("395 kg") are all cleaned up automatically before the data is used.
It never guesses. If a value is missing or unreadable, it's left blank and reported — never estimated or invented. The dashboard always reflects actual plant data only.
This extraction module is built to be upgraded with a Claude AI-powered backend later without changing anything else in the dashboard — until that's connected, this reliable rule-based engine handles every upload.

Alerts & Severity Levels

How to read the Alerts & Insights page — every alert is built the same way.
Each active alert shows the same fields, always in this order: Parameter and Current vs Target in the title · Priority Level · Magnitude of Deviation — the exact size of the gap, in both % and real units · Supporting Evidence — comparison against yesterday, the 7-day average, and the monthly average · Root Cause — a specific, data-backed explanation, including a link to raw material quality where the evidence supports it · Operational Impact · Corrective Action — what to do, and which department should do it.
Priority levels:
WITHIN TARGET LOW MEDIUM HIGH CRITICAL
LOW means a parameter has just stepped outside its target — worth noting, rarely worth immediate action on its own. Within Target isn't an alert at all; it's shown separately under Positive Indicators.
"CONTINUOUS DETERIORATION" replaces the usual priority tag when a parameter has been getting worse for three or more days in a row — it's a trend warning, not just a single bad reading, and usually deserves priority attention over a one-off spike regardless of its current tier.
The Daily Furnace Diagnosis card on the same page summarizes all of this into one view: Overall Condition, Primary and Secondary Concern, Best Performing Parameter, Critical Parameter, and Immediate Action Required.

Validation Errors

Every upload is checked before it's accepted — here's what that looks like.
Missing or invalid value (e.g. a cell reads "N/A" or text where a number is expected) → that one field is left blank; the row is otherwise kept.
Unrecognized date format → that row is skipped and reported, so it never gets a wrong date.
Duplicate date within the same file → the later occurrence is kept, and it's noted in the report.
File structure not recognized at all (e.g. wrong template, no Date column found) → the whole file is flagged Manual Entry Required with the specific reason, and nothing is merged — the dashboard is never left in a half-updated state.
After every upload, click 📋 View detailed report in the status panel to see the exact row, field, issue, and what the dashboard did about it — nothing is hidden.

Best Practices

A few habits that keep the dashboard reliable.
Upload the original DPR/MIS file — avoid manually reformatting it before uploading.
Keep one person responsible for shared updates each day, to avoid conflicting edits landing at the same time.
Always check the validation report after an upload before relying on the numbers — a successful merge and a clean merge aren't always the same thing.
Where possible, keep column headers consistent across months — it helps the extraction engine match them reliably every time.
Treat Alerts & Insights as the starting point for investigation, not the final word — it points to evidence and a likely cause, but on-the-ground judgment always has the final say.
Before presenting the dashboard to others, confirm the data range indicator shows the date you expect — it's the fastest way to catch stale data.

Data Management
DPR / MIS Upload Center · Data Ingestion & Validation
ℹ️ Extraction Methodology: Uploaded Excel files are scanned for a header row matching known Blast Furnace / Raw Material parameter names — there are no fixed cell references. Matched columns are extracted; unrecognized columns are ignored. Every upload — manual, shared, or linked — is validated (date format, mandatory fields, duplicate records, numeric consistency) before it is merged into the live dataset. Files that cannot be reliably interpreted are flagged Manual Entry Required with the specific reason, and never crash the dashboard. The Shared Master Dataset is fetched from this dashboard's own GitHub repository on every page load, so it's the same on any computer. Live Excel Sync (Chrome/Edge only) additionally lets this specific computer re-check a locally linked file every few seconds and pull in changes automatically after it's saved in Excel — that link only ever works on the device that created it.
